Creating a Time Tracking App With Meteor

Apr 1st, 2014 | Comments

This is a tutorial for developers meeting Meteor for the first time. We’ll develop a simple time tracking application.

Since this tutorial is to learn how to develop a web app with Meteor, we won’t build a list of requirements upfront and implement all features taking into account how each piece of code can be extended to implement other future functionalities. We’ll focus on a feature at a time instead, writing the code that’s strictly required to make that feature work. Let’s say it’s a TDD approach, but without the test part, or Feature-driven development if you prefer.

What’s meteor

If you are reading this tutorial, I presume you already know what meteor is. The official website defines it as

an open-source platform for building top-quality web apps in a fraction of the time, whether you’re an expert developer or just getting started

My own definition istead is:

Meteor is the evolution of web development, where the server joins the client, shortcutting their distance, and making reactivity the most predonimant verb

The toolset

We just need a few tools:

  • meteor
  • an editor (pick your favorite one)
  • a browser
